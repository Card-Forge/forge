  Summary

  I extended the existing MTGGoldfish scraper into a full .dck importer so the
  self-play / gauntlet harness can run real metagame decks instead of stand-ins.

  app/knowledge/scraper.py — added the deck-download → .dck pipeline, building
  on the existing archetype_links() map:
  - archetype_deck_id(page_url) — follows an archetype page to its
  representative deck id (regex on /deck/download/<id>, falling back to the
  canonical /deck/<id> link). Fail-soft → None.
  - fetch_deck_download(deck_id) — GETs /deck/download/<id>. Fail-soft → "".
  - parse_goldfish_decklist(text) — pure parser handling both export shapes:
  plain .txt (blank line splits main/sideboard) and Arena format (Deck/Sideboard
   headers + (SET) 123 tails, which are stripped; Companion/Maybeboard ignored).
  - decklist_to_dck(name, main, side) — renders Forge .dck text with bare card
  names so Forge resolves a default printing (the most robust name mapping).
  - fetch_deck_as_dck(deck_id, name) — convenience that chains the three.

  Network helpers fail soft (matching the module's existing convention); the
  parse/convert helpers are pure and offline.

  scripts/import_goldfish_decks.py — new CLI that walks a format's archetypes
  and writes <out>/<format>/<archetype-slug>.dck (default selfplay/decks/),
  reusing slugify for filenames. Supports --archetype, --limit, --out, --force,
  --sleep (2s default, polite to the source), skips existing files, and is
  best-effort per archetype.

  tests/test_scraper_dck.py — offline parser/formatter tests covering both
  export shapes, set-code stripping, companion exclusion, and the .dck
  round-trip (an isolated-parser test in the spirit of test_clean_html.py, not
  new wiring).

  Lint (ruff), format (black), and the full suite (207 passed) are all green.

  Usage:
  python scripts/import_goldfish_decks.py modern --limit 8
  # → selfplay/decks/modern/boros-energy.dck, ...
  forge selfplay -config goldfish -p1 selfplay/decks/modern/boros-energy.dck -p2
   60-islands.dck -n 50 -out runs/boros.jsonl

  One thing to flag: I haven't verified against MTGGoldfish's live markup (no
  network here), so the deck-id selectors and download format are based on their
   known/historical shapes. A first real run with --limit 1 is the right smoke
  test before bulk-importing.
