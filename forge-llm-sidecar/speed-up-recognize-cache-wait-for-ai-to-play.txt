● Diagnosis confirmed and patched

  Root cause: Android 9+ blocks cleartext (plain HTTP) traffic from apps by
  default. Your manifest had targetSdkVersion=35 with no exemption, so Forge's
  Java HttpURLConnection was being silently denied at the OS layer — connections
   never reached the sidecar. The dashboard view worked because Chrome is a
  different app with its own policy.

  The proof from your log:
  - attach() ran correctly: enabled=true sysprop=true
  - /health check failed at attach: did not respond to /health
  - 4288 POST /recognize attempts, ALL returned empty Optional → silent
  IOExceptions logged at DEBUG (not visible in your log file)

  Patch applied (two files):
  1. forge-gui-android/src/main/AndroidManifest.xml — added
  android:networkSecurityConfig="@xml/network_security_config"
  2. forge-gui-android/res/xml/network_security_config.xml (new) — allows
  cleartext only to Tailscale IPs (your sidecar 100.110.52.69, plus loopback for
   testing). Everything else still requires HTTPS, preserving Android's default
  safety.

  Next steps

  cd /home/lou/Development/forge-1
  mvn -pl forge-gui-android -am -DskipTests package
  # install the new APK on your phone (adb / sideload)

  After install:
  - Open Forge → start a new constructed match against the AI
  - Watch the sidecar log (tail -F 
  /home/lou/Development/forge-1/forge-llm-sidecar/logs/sidecar.log)
  - You should immediately see POST /identify-own-archetype followed by POST 
  /recognize once the AI's opponent (you) plays a card

  If for some reason you still don't see traffic after this rebuild, the next
  thing to check is whether the sidecar's localhost resolution is somehow
  specific (we shouldn't have that issue since you're using a direct IP, but
  worth flagging).
