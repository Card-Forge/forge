     Add a "Enable Deck Recognition for all AI profiles" preference

     Context

     The branch feature/llm-deck-recognition adds an LLM-powered
     deck-recognition
     feature. Today it can only be turned on per-AI-profile (the
     DECK_RECOGNITION_ENABLE
     property inside each .ai file, which currently defaults to false and is set
      in
     none of the shipped profiles) or via the -Dforge.ai.deckRecognition=true
     JVM
     system property. There is no in-app way for a user to switch it on.

     This change adds a user-facing checkbox in the Preferences screen that
     enables deck
     recognition globally — for every AI profile — without editing profile files
      or
     passing JVM flags.

     Approach

     DeckRecognitionFeature.isEnabled() (in module forge-ai) already treats the
     forge.ai.deckRecognition system property as a global override that wins
     over any
     profile setting. Since forge-ai cannot see ForgePreferences (it lives in
     forge-gui), we bridge the new preference to that existing system property.
     No
     change to forge-ai is required.

     1. Declare the preference

     forge-gui/src/main/java/forge/localinstance/properties/ForgePreferences.jav
     a
     - Add a new enum value near UI_CURRENT_AI_PROFILE (line 129):
     UI_ENABLE_DECK_RECOGNITION ("false"),

     2. Bridge the preference to the system property

     forge-gui/src/main/java/forge/player/GamePlayerUtil.java —
     createAiPlayer(...)
     (lines 69-96). This static method runs for every AI player created, so it
     always
     reflects the current preference. Before/after building the player, sync the
      flag:
     System.setProperty(DeckRecognitionFeature.SYS_PROP,
         String.valueOf(FModel.getPreferences().getPrefBoolean(FPref.UI_ENABLE_D
     ECK_RECOGNITION)));
     DeckRecognitionFeature.SYS_PROP is the public constant
     "forge.ai.deckRecognition"
     (forge-ai/.../llm/DeckRecognitionFeature.java:15).
     DeckRecognitionFeature.isEnabled()
     reads it via Boolean.getBoolean(SYS_PROP) first, so this transparently
     enables the
     feature for whichever profile the AI ends up using. Setting it explicitly
     to
     "false" when unchecked keeps it self-correcting.

     3. Localization strings

     forge-gui/res/languages/en-US.properties — add next to the cbEnableAICheats
      /
     nlEnableAICheats entries (lines 68 and 163):
     cbEnableDeckRecognition=Enable AI Deck Recognition
     nlEnableDeckRecognition=Let the AI use the LLM sidecar to identify your
     deck archetype during a game (applies to all AI profiles; requires the
     sidecar service running).

     4. Mobile UI (libGDX)

     forge-gui-mobile/src/forge/screens/settings/SettingsPage.java — in the
     Gameplay
     group (group index 1), alongside the existing UI_ENABLE_AI_CHEATS entry
     (lines 196-198), add:
     lstSettings.addItem(new BooleanSetting(FPref.UI_ENABLE_DECK_RECOGNITION,
         Forge.getLocalizer().getMessage("cbEnableDeckRecognition"),
         Forge.getLocalizer().getMessage("nlEnableDeckRecognition")), 1);
     BooleanSetting already handles toggle + setPref + save.

     5. Desktop UI (Swing)

     forge-gui-desktop/src/main/java/forge/screens/home/settings/CSubmenuPrefere

     Critical files

     - forge-gui/src/main/java/forge/localinstance/properties/ForgePreferences.j
     ava — new FPref

     Verification

     1. Build: mvn -q -pl forge-gui,forge-gui-desktop,forge-gui-mobile -am 
     compile.
     2. Desktop: launch Forge, open Settings → Preferences, confirm the new
     "Enable AI Deck Recognition" checkbox appears under Gameplay; toggle it and
     confirm UI_ENABLE_DECK_RECOGNITION=true is written to the forge.preferences
     file in the user prefs dir.
     3. Mobile: launch the mobile build, open Settings, confirm the checkbox
     renders in
     the Gameplay group and persists across restarts.
     4. End-to-end: with the LLM sidecar running (forge-llm-sidecar, default

     the
     game log — confirming the system-property bridge enabled the feature
     without
     editing any .ai profile. Uncheck it and confirm a new match produces no
     recognition log entries.
