● Job is still running. Here's a clear diagnosis of both problems:

  ---
  Problem 1: Boros Energy — page fetched (1.3MB) but only 40 chars extracted

  parse_sections walks article.descendants looking for <h2> and <p> tags. The
  /decks/boros-energy/ URL (where the redirect landed) uses a different page
  template — likely <div class="entry-content"> with nested divs, not a semantic
   <article> element with tidy <h2> sections. So the H2 headings never fire, and
   only one stray <p> gets captured.

  Problem 2: Ruby Storm / Jeskai Blink — genuine 404s across all 4 candidates

  The 4 URL patterns tried are:
  mtg-{slug}-modern-deck/
  {slug}-modern/
  mtg-{slug}-deck/
  {slug}/
  Draftsim just doesn't have pages for these under those patterns. Missing
  candidates like {slug}-mtg/ (the redirect target for Affinity), decks/{slug}/
  (Boros Energy's actual path), {slug}-deck/.

  ---
  Two fixes:

  Fix A — candidate_urls: add 3 more patterns
  f"https://draftsim.com/{slug}-mtg/",        # affinity redirect target
  f"https://draftsim.com/decks/{slug}/",       # boros-energy redirect target
  f"https://draftsim.com/{slug}-deck/",        # common variant

  Fix B — parse_sections: raw text fallback
  After parsing, if primer_to_prompt_text(sections) returns < 200 chars,
  re-parse the HTML with a broader selector — grab all <p> tags from
  div.entry-content (or body), strip ad/nav noise, and dump them into the
  "intro" key. This recovers the Boros Energy case.

  On making the parser less deterministic: Yes, this is the right call. Each
  provider uses different heading structures, so a rigid H2-section parser will
  miss most content. The better approach is: fetch the HTML, strip boilerplate
  (nav/footer/ads), dump the cleaned text to the LLM and ask it to extract the
  sections we care about. We lose nothing by doing this — the LLM is already
  being called per archetype anyway.

  Proposed approach:
  1. Phase 1 URL discovery: try CardsRealm → CK Blog → PlayingMTG → Mystical
  Teachings → Draftsim (non-/decks/) in order, confirm each returns a real
  strategy page (not a decklist)
  2. Phase 2: fetch confirmed URLs, pass raw cleaned text to LLM with a prompt
  like "extract game plan, mulligan advice, matchup notes, sideboard guide from
  this article"
  3. Fall back to LLM general knowledge only if no provider had a page
